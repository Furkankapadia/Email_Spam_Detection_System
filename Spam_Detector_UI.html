Spam Email Detector

A machine learning project that classifies SMS/email messages as spam or ham using Natural Language Processing and Naive Bayes classification. Built with a 97.31% accuracy on the UCI SMS Spam Collection dataset.

🐍 Python 📊 Pandas 🔢 NumPy 🤖 Scikit-learn 📝 NLTK 🔤 TF-IDF 📐 MultinomialNB 💾 Pickle
📂

Dataset

Uses the spam.csv dataset with 5,572 messages (4,825 ham + 747 spam). The dataset comes from the UCI Machine Learning Repository's SMS Spam Collection.

🧠

Algorithm

Multinomial Naive Bayes is used as the classifier — ideal for discrete features like word counts / TF-IDF scores. Known for high speed and strong text classification performance.

🔤

NLP Pipeline

Text is lowercased, cleaned via regex, tokenized, stop words removed using NLTK, and stemmed using PorterStemmer before vectorization.

📋 Project Pipeline — Step by Step
1
Load & Prepare Dataset

Load spam.csv using Pandas, drop unnamed columns, rename to label and message. Dataset has 5,572 rows with 0 missing values.

2
Text Preprocessing

Convert to lowercase → remove non-alphabetic characters via regex → split into tokens → remove English stopwords (NLTK) → apply Porter Stemming to reduce words to their root form.

3
Feature Extraction — TF-IDF

Use TfidfVectorizer(max_features=3000) to convert processed text into a 5572×3000 numeric feature matrix. Labels encoded: ham=0, spam=1.

4
Train/Test Split

80/20 stratified split ensures class proportions are preserved. Result: 4,457 training samples and 1,115 test samples.

5
Model Training

Train MultinomialNB() on the training set. Naive Bayes assumes feature independence and works well with high-dimensional sparse TF-IDF vectors.

6
Evaluation

Accuracy: 97.31% | Precision: 99.17% | Recall: 80.54% | F1: 88.89%. Confusion matrix shows only 1 false positive and 29 false negatives.

7
Save & Deploy

Model and vectorizer serialized with pickle to spam_model.pkl and vectorizer.pkl for future inference.
